# 工具链

## Hooks（自动钩子）

Hooks 在 `.claude/settings.json` 中配置，在特定事件时自动触发。

| Hook | 触发事件 | 匹配条件 | 作用 |
|------|---------|---------|------|
| `validate-commit.sh` | PreToolUse (Bash) | `git commit` 命令 | 验证设计文档节完整性、JSON 数据文件、硬编码值、TODO 格式 |
| `validate-push.sh` | PreToolUse (Bash) | `git push` 命令 | 推送到保护分支（main/develop）时警告 |
| `validate-assets.sh` | PostToolUse (Write/Edit) | `assets/` 目录文件 | 检查命名规范和 JSON 有效性 |
| `session-start.sh` | SessionStart | 会话开始 | 加载 Sprint 上下文、里程碑、Git 活动记录；检测并预览会话状态文件 |
| `detect-gaps.sh` | SessionStart | 会话开始 | 检测空项目（建议 `/start`）和缺失文档 |
| `pre-compact.sh` | PreCompact | 上下文压缩前 | 将会话状态转储到对话中，确保压缩后可恢复 |
| `post-compact.sh` | PostCompact | 压缩后 | 提醒从 `active.md` 恢复会话状态 |
| `notify.sh` | Notification | 通知事件 | Windows 系统托盘通知 |
| `session-stop.sh` | Stop | 会话结束 | 总结工作成果，更新会话日志 |
| `log-agent.sh` | SubagentStart | Agent 启动 | 审计日志：记录子 Agent 调用 |
| `log-agent-stop.sh` | SubagentStop | Agent 停止 | 审计日志：完成子 Agent 记录 |
| `validate-skill-change.sh` | PostToolUse (Write/Edit) | Skill 文件修改 | 提醒运行 `/skill-test` 验证 |

## Rules（路径规则）

规则文件放在 `.claude/rules/` 中，编辑匹配路径的文件时自动生效。

| 规则文件 | 匹配路径 | 强制内容 |
|---------|---------|---------|
| `gameplay-code.md` | `src/gameplay/**` | 数据驱动值、使用 delta time、不引用 UI |
| `engine-code.md` | `src/core/**` | 热路径零分配、线程安全、API 稳定性 |
| `ai-code.md` | `src/ai/**` | 性能预算、可调试性、数据驱动参数 |
| `network-code.md` | `src/networking/**` | 服务器权威、版本化消息、安全性 |
| `ui-code.md` | `src/ui/**` | 不持有游戏状态、本地化就绪、无障碍 |
| `design-docs.md` | `design/gdd/**` | 必须包含 8 个章节、公式格式、边界情况 |
| `narrative.md` | `design/narrative/**` | 设定一致性、角色语调、设定正典层级 |
| `data-files.md` | `assets/data/**` | JSON 有效性、命名规范、Schema 规则 |
| `test-standards.md` | `tests/**` | 测试命名、覆盖要求、夹具模式 |
| `prototype-code.md` | `prototypes/**` | 放宽标准、README 必需、假设文档化 |
| `shader-code.md` | `assets/shaders/**` | 命名规范、性能目标、跨平台规则 |

## 上下文管理

### 核心原则

**文件是记忆，对话不是。** 对话会被压缩或丢失，文件在磁盘上持久存在。

### 会话状态文件

维护 `production/session-state/active.md` 作为活跃检查点：

```markdown
<!-- STATUS -->
Epic: Combat System
Feature: Melee Combat
Task: Implement hitbox detection
<!-- /STATUS -->
```

在以下情况更新：
- 设计章节审批并写入文件后
- 架构决策做出后
- 实现里程碑达成后
- 测试结果获得后

### 增量文件写入

创建多节文档时：

1. 立即创建骨架文件（所有章节标题，空内容）
2. 逐节讨论并起草
3. 每节审批后立即写入文件
4. 写入后，之前的讨论可安全压缩

**效果**：上下文窗口只保留当前章节讨论（~3-5k tokens），而非整个文档历史（~30-50k tokens）。

### 主动压缩

- **在 ~60-70% 上下文使用率时主动压缩**，而非等到上限
- **不相关任务间用 `/clear`**
- **聚焦压缩**：`/compact Focus on [当前任务] — 1-3 节已写入文件，正在处理第 4 节`

### 压缩后保留信息

压缩时确保以下信息在摘要中保留：

- `production/session-state/active.md` 的引用
- 本次会话修改的文件列表和用途
- 做出的架构决策及其原因
- 活跃 Sprint 任务和当前状态
- Agent 调用及其结果
- 测试结果（通过/失败计数，具体失败项）
- 未解决的阻塞或等待用户的问题

### 会话崩溃恢复

如果会话中断或需要新会话继续：

1. `session-start.sh` 自动检测并预览 `active.md`
2. 读取完整状态文件获取上下文
3. 读取正在处理的文件
4. 从下一个未完成节或任务继续

## 模型分层

| 层级 | 模型 | 适用场景 |
|------|------|---------|
| **Haiku** | `claude-haiku-4-5-20251001` | 只读状态检查、格式化、简单查询 — 无需创意判断 |
| **Sonnet** | `claude-sonnet-4-6` | 实现、设计编写、单系统分析 — 大多数工作的默认选择 |
| **Opus** | `claude-opus-4-6` | 多文档综合、高风险阶段门禁、跨系统整体审查 |

**Haiku 模型的 Skill**：`/help`、`/sprint-status`、`/story-readiness`、`/scope-check`、`/project-stage-detect`、`/changelog`、`/patch-notes`、`/onboard`

**Opus 模型的 Skill**：`/review-all-gdds`、`/architecture-review`、`/gate-check`

其余 Skill 默认使用 Sonnet。
